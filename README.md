## 🛠 Использованные инструменты
- **Язык программирования:** Python 3
- **Фреймворк:** `pytest` (для структурирования и запуска тестов)
- **HTTP-клиент:** библиотека `requests` (для отправки запросов к API)

## 🚀 Инструкция по запуску

1. Убедитесь, что у вас установлен Python (версии 3.8+).
2. Склонируйте репозиторий и перейдите в папку с проектом.
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите тесты командой:
   ```bash
   pytest test_api.py -v
   ```
   *Флаг `-v` (verbose) позволяет увидеть подробный вывод по каждому пройденному тесту.*

## 📋 Краткий список реализованных проверок
В рамках задания протестирован флоу авторизации и работы с корзинами. Реализованы следующие сценарии:

**Авторизация (Auth):**
1. Успешная авторизация по валидным логину и паролю.
2. Неуспешная авторизация (неверный пароль) -> Ожидаем 400 статус.
3. Получение данных текущего пользователя `GET /auth/me` с передачей Bearer-токена.
4. Попытка запросить `GET /auth/me` без токена -> Ожидаем 401 статус.

**Корзины (Carts):**
5. Получение списка корзин конкретного пользователя (ID берется из успешной авторизации).
6. Получение данных корзины по её ID.
7. Создание новой корзины (добавление товаров).
8. Обновление корзины (изменение quantity товара) через метод PUT.
9. Удаление корзины (проверка флага `isDeleted: true`).
10. Негативный сценарий: Поиск несуществующей корзины по невалидному ID -> Ожидаем 404 статус.

**Особенности работы API:**
Тесты успешно проходят. Важно отметить, что DummyJSON имитирует методы POST/PUT/DELETE: сервер возвращает успешные ответы и измененное тело, но в реальности база данных не обновляется (что является стандартным поведением данного мок-сервера). Тесты написаны с учетом этого поведения.
