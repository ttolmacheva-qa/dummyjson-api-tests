## 🛠 Использованные инструменты
- **Язык программирования:** Python 3
- **Тестовый фреймворк:** `pytest`
- **HTTP-клиент:** библиотека `requests`
- **Отчетность:** `pytest-html` (генерация интерактивного HTML-отчета)
- **CI/CD:** GitHub Actions (автоматический запуск тестов при каждом push-событии)

## 🚀 Инструкция по запуску

1. Убедитесь, что у вас установлен Python (версии 3.8+).
2. Склонируйте репозиторий и перейдите в папку с проектом.
3. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для macOS/Linux
   # или
   venv\Scripts\activate     # Для Windows (PowerShell)
   ```
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
5. Запустите тесты с генерацией HTML-отчета:
   ```bash
   pytest test_api.py -v --html=report.html --self-contained-html
   ```
   *Сгенерированный файл `report.html` можно открыть в браузере для просмотра интерактивного отчета.*

## 📋 Краткий список реализованных проверок
В рамках задания протестирован флоу авторизации и работы с корзинами. Реализованы следующие сценарии (в файле `test_api.py`):

**Авторизация (Auth):**
* **1.** Успешная авторизация по валидным логину и паролю (`test_login_success`).
* **2.** Неуспешная авторизация с неверным паролем -> Ожидаем 400 статус (`test_login_invalid_password`).
* **3.** Получение данных текущего пользователя с токеном в заголовках (`test_get_current_user_with_token`).
* **4.** Попытка запросить данные без токена -> Ожидаем 401 статус (`test_get_current_user_without_token`).

**Корзины (Carts):**
* **5.** Получение списка корзин конкретного пользователя (`test_get_user_carts`).
* **6.** Получение данных корзины по её ID (`test_get_cart_by_id`).
* **7.** Создание новой корзины (`test_add_cart`).
* **8.** Обновление корзины методом PUT (`test_update_cart`).
* **9.** Удаление корзины методом DELETE (`test_delete_cart`).
* **10.** Негативный сценарий: Поиск несуществующей корзины по невалидному ID -> Ожидаем 404 статус (`test_get_cart_not_found`).

**Особенности работы API:**
Тесты успешно проходят. Важно отметить, что DummyJSON имитирует методы POST/PUT/DELETE: сервер возвращает успешные ответы и измененное тело, но в реальности база данных не обновляется (что является стандартным поведением данного мок-сервера). Тесты написаны с учетом этого поведения.
